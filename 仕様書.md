# 一級建築士製図試験 時間管理ツール 仕様書

作成日：2026-03-25　最終更新：2026-03-26

---

## 1. 概要

一級建築士製図試験（6時間30分）の練習における時間管理・記録・分析ツール。
HTML単体ファイルで動作し、PC・タブレット・スマホのすべてに対応する。

---

## 2. 動作環境

- **形式**：HTML（単一ファイル。CSS・JS内包）
- **対応端末**：PC / タブレット / スマートフォン（レスポンシブ対応）
- **通信**：オンライン前提（Chart.js を CDN で読み込み）
- **データ保存**：ブラウザの LocalStorage に保存（サーバー不要）

---

## 3. 試験モード

以下の4モードを選択して開始できる。

| モード | 内容 | 目標時間 |
|--------|------|----------|
| フルモード | エスキス＋記述＋作図（全工程） | 390分 |
| エスキスのみ | エスキス工程のみ計測 | 150分 |
| 記述のみ | 記述工程のみ計測 | 60分 |
| 作図のみ | 作図工程のみ計測 | 180分 |

## 5. タイマー機能

- **方式**：カウントアップ（0:00:00 から計測）
- **操作**：
  - 開始ボタン → タイマースタート
  - **ラップボタン**（ワンタップで現在の工程時間を記録し、次の工程へ進む）
  - 一時停止 / 再開ボタン
  - 終了ボタン
- **キーボードショートカット**（PC）：
  - Enter：次の工程へ（ラップ）
  - Space：一時停止 / 再開

### 一時停止の挙動

- 一時停止中は**累計タイマー・工程タイマーの両方を完全停止**
- 再開ボタンで停止した時点からそのまま計測を再開

### 表示レイアウト（優先順位）

```
┌─────────────────────────────┐
│   累計タイム（最大・最目立つ）      │
│        0 : 00 : 00           │  ← 画面の中心・最大フォント
├─────────────────────────────┤
│  現在の工程名                   │
│  工程経過時間  /  目標時間        │  ← 目標超過で赤表示
├─────────────────────────────┤
│  次の工程（薄く表示）             │
├─────────────────────────────┤
│  [ ラップ（次の工程へ） ]         │  ← 大きなボタン
│  [ 一時停止 ]  [ 終了 ]         │
└─────────────────────────────┘
```

- **累計タイム**：画面の中央に最大フォントで表示（一番目立つ要素）
- **工程経過時間**：現在工程の経過時間を目標時間と並べて表示
- 目標時間を超えた場合は工程時間の表示色を赤に変える（アラート音なし）
- 計測中は不要なUIを非表示にし、タイマーに集中できる画面構成
- ラップボタンは大きく押しやすいサイズ（ワンタップ操作を前提）

## 6. セッション記録

### 6-1. セッション開始時の入力

- **課題名**（例：「課題No.3 福祉施設」）※過去に入力した課題名をサジェスチョン表示
- **モード選択**（フル / エスキスのみ / 記述のみ / 作図のみ）
- **メモ**（任意・自由記述）

### 6-2. 計測中の操作

- ラップボタンのワンタップのみ（評価・メモの入力は行わない）

### 6-3. セッション終了後 → 振り返り画面

全工程が完了、または途中で「終了」を押した時点で**振り返り画面**に遷移する。

振り返り画面では各工程について以下を表示・入力できる：

| 表示項目 | 内容 |
|---------|------|
| 工程名 | ステップ名 |
| 今回のラップタイム | 実測値 |
| 目標時間 | 基準値 |
| **過去平均ラップタイム** | 同モード・同工程の履歴平均（初回は「―」） |

| 入力項目 | 内容 |
|---------|------|
| **評価** | 👍 Good / 👎 Bad / ―（未評価）の3択 |
| **メモ** | 自由記述（例：「外壁迷った」「思ったより早かった」） |

- 評価のデフォルトは目標時間との比較で自動設定（目標以内→Good、超過→Bad）。手動で変更可能
- 未評価は分析時の集計対象外
- 今回のラップタイムと過去平均を並べて比較できる
- 振り返り入力後「保存」ボタンでセッションデータを確定

### 6-4. 保存データ

- 実施日時
- 課題名・モード
- 各工程：ラップ時間（実績）・目標時間・評価（Good/Bad/未評価）・メモ
- 合計時間
- セッション全体メモ

## 7. 履歴・分析機能

### 7-1. 履歴一覧

- 過去のセッションを日付順に一覧表示
- 課題名・合計時間・モードをカード形式で表示
- クリックで詳細（工程ごとのタイム・評価・メモ）を展開表示
- セッションの削除機能

### 7-2. グラフ・分析

- **棒グラフ**：セッションごとの合計時間の推移（成長グラフ）
- **積み上げ棒グラフ**：工程ごとの時間内訳の変化
- **工程別比較**：各工程の平均・最速・最遅を表示
- **Good/Bad 分析**：各工程の Good 率を表示（どの工程が安定しているか一目でわかる）
- **Bad 工程の傾向**：Bad が多い工程をハイライト表示（課題の発見に活用）
- 履歴詳細には工程ごとの評価（👍/👎）とメモを表示
- フィルタ：モード別・課題名別で絞り込み可能

### 7-3. データ管理

#### バックアップ（JSONファイル）

- **エクスポート**：ボタンを押すと `.json` ファイルをダウンロード
- **インポート**：ファイルを選択して読み込み（端末間の引き継ぎ・復元用）
- ファイル名には日付を自動付与（例：`seizu_backup_20260325.json`）

#### 分析用エクスポート（CSV形式 → Googleスプレッドシート対応）

- セッションデータを**CSV形式**でダウンロードできる
- 1行 ＝ 1セッション × 1工程（フラット構造）

```
CSVカラム例：
実施日, 課題名, モード, 工程番号, 工程名, 目標時間(分), 実績時間(分), 評価, メモ, 合計時間(分)
```

- 時間は分単位（実績・合計は秒を切り上げ）
- Googleスプレッドシートで直接開ける（ドラッグ＆ドロップまたはインポート）
- スプレッドシート上でピボットテーブル・独自グラフ・フィルタ等の高度な分析が可能
- 文字コードは UTF-8（BOM付き）で日本語の文字化けを防止

#### データリセット

- **全削除**：確認ダイアログ付きで全データをリセット

## 9. 画面構成

```
ホーム画面
  ├→ 新規セッション → セッション設定画面 → タイマー画面 → 振り返り画面 → 保存完了
  ├→ 履歴一覧 → セッション詳細
  ├→ 分析（グラフ）
  └→ 設定（工程カスタマイズ / バックアップ / データ管理）
```

| 画面 | 概要 |
|------|------|
| ホーム | モード別練習回数・ベストタイム・前回タイム表示。新規セッション・履歴・分析・設定へのメニュー |
| セッション設定 | 課題名・モード・メモを入力して「開始」 |
| タイマー | 計測画面。累計タイム・工程タイム・ラップボタンのみの集中レイアウト |
| 振り返り | セッション終了後。各工程の実績 vs 目標 vs 平均を比較し、評価・メモを入力 |
| 履歴一覧 | 過去セッションのカード一覧 |
| セッション詳細 | 1セッションの全工程タイム・評価・メモを表示 |
| 分析 | グラフ表示。フィルタ付き |
| 設定 | 工程カスタマイズ・テーマカラー・バックアップ・エクスポート・データ管理 |

## 10. UI / UX 方針

- **シンプルで集中できる**デザイン（計測中は余計な情報を排除）
- 現在の工程を大きく表示
- ダークモード（目に優しい）
- ボタンは大きく押しやすい（タブレット・スマホ操作を考慮）
- 縦・横どちらのレイアウトでも崩れないレスポンシブ対応
- **テーマカラーのカスタマイズ**：8色のプリセット＋カスタムカラーピッカーで設定可能

## 11. 技術仕様

- **HTML5 / CSS3 / JavaScript（Vanilla JS）**
- 外部ライブラリ：Chart.js（グラフ描画） CDN 読み込み
- LocalStorage にすべてのデータを保存
- 単一HTMLファイルで完結（配布・共有が容易）

## 12. セキュリティ

| 対策 | 実装内容 |
|------|---------|
| **XSS防止** | ユーザー入力（課題名・メモ等）の表示は `textContent` を使用。`innerHTML` に直接代入しない |
| **CDN改ざん検知** | Chart.js の `<script>` タグに `integrity`（SRI ハッシュ）と `crossorigin="anonymous"` を付与 |
| **インポート検証** | JSONインポート時にスキーマ検証を実施。想定外のキー・型・サイズのデータは無視・除外 |
| **CSVインジェクション防止** | CSV出力時、セル値の先頭が `=`, `+`, `-`, `@` の場合はシングルクォートでエスケープ |
